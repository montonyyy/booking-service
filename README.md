# Сервис для управления бронированиями

Этот сервис предоставляет возможность создавать, удалять, изменять и просматривать бронирования помещений/событий и прослушивать любое их изменение (с помощью NOTIFY/LISTEN).

При развертывании данного сервиса запускается сервер, который прослушивает tcp адрес, и вследствие взаимодействует с базой данных. Аналогично запускается tg BotAPI.

---

## Работа с сервисом  
Для начала работы необходимо задать все переменные окружения, которые лежат в .env (.env.example):  
1. *CONN_STRING* - адрес подключения к базе данных  
2. *BOT_TOKEN*   - токен tg бота (опционально; если не указать - функционал тг бота работать не будет)  
3. *ADMIN_ID*    - айди пользователя, который будет иметь право пользоваться ботом  
4. *SERVER_PORT* - порт, которого будет прослушивать сервер  

### Взаимодействие с API
1. Получение данных с БД --> Метод GET на эдпоинте /booking  
2. Отправка данных в БД  --> Метод POST с телом, содержащее place_id, user_name, user_phone, start_time, end_time, на эдпоинте /booking  
3. Удаление данных с БД  --> Метод DELETE с телом, содержащее id, на эндпоинте /booking  
4. Изменение данных в БД --> Метод PATCH с телом, содержащее id, start_time, end_time, на эндпоинте /booking  

### Взаимодействие с tg BotAPI
1. Получение данных с БД --> /list  
2. Отправка данных в БД  --> /add  
3. Удаление данных с БД  --> /del  
4. Изменение данных в БД --> /rep
