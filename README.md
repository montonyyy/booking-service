# Сервис для управления бронированиями

Данный сервис предоставляет возможность создавать, удалять и просматривать бронирования помещений/событий.

При развертывании данного сервиса запускается сервер, который прослушивает tcp адрес, и вследствие взаимодействует с базой данных. Аналогично запускается tg BotAPI.

1. Работа с сервисом
Для начала работы необходимо задать все переменные окружения, которые лежат в .env (.env.example). *CONN_STRING* - адрес подключения к базе данных; *BOT_TOKEN* - токен tg бота (опционально; если не указать - функционал тг бота работать не будет); *ADMIN_ID* - айди пользователя, который будет иметь права пользоваться ботом. *SERVER_PORT* - порт, который будет прослушивать сервер.

2. Запросы для сервера
Метод GET на эдпоинте /booking --> получение данных с бд
Метод POST с json телом на эдпоинте /booking --> отправка данных в бд
Метод DELETE с json телом на эндпоинте /booking --> удаление данных с бд

3. Запросы для tg бота
/list --> получение данных с бд
/add --> отправка данных в бд
/del --> удаление данных с бд

#### Важно! Это учебный проект. Он ещё разрабатывается, и в нём множество компромиссов.
